### **1-Digit Decimal Up Counter Using D Flip-Flops**



**## Introduction**



This project focuses on the design and implementation of a 1-digit decimal up counter using 74HC74 D Flip-Flop integrated circuits. The counter is capable of counting sequentially from 0 to 9 and displaying the output on a seven-segment display. Once the count reaches 9, the system automatically resets and starts counting again from 0.



The project was developed as part of the IE2044 – System Modeling and Prototyping module and combines digital electronics design, circuit simulation, PCB development, and hardware prototyping.



**## Project Objectives**



The primary objective of this project is to understand the practical implementation of digital counters using D Flip-Flops and logic gates. The project also aims to provide experience in electronic circuit design, simulation, PCB development, and enclosure modeling.



**Specific objectives include:**



\- Designing a decimal up counter using D Flip-Flops

\- Implementing automatic reset functionality

\- Displaying count values on a seven-segment display

\- Simulating the circuit using Proteus

\- Designing a PCB using EasyEDA

\- Creating a 3D enclosure using Autodesk Fusion 360

\- Testing and validating the final design



**## System Description**



The counter is implemented using 74HC74 D Flip-Flop ICs connected in an asynchronous ripple-counter configuration. Logic gates are used to detect the decimal value 9 and generate a reset signal that returns the counter to 0.



The binary output of the counter is connected to a BCD-to-Seven-Segment Decoder which converts the binary value into signals suitable for driving a common anode seven-segment display.



Additional push buttons are included to support manual reset and pause/resume operations.

**## Design and Implementation Process**



The project began with the design of the digital logic circuit using D Flip-Flops. The circuit was first simulated in Proteus to verify correct counting operation and reset functionality.



Once the simulation was validated, the schematic was transferred to EasyEDA for PCB development. The PCB was designed by considering component placement, routing efficiency, and manufacturing requirements.



Finally, a protective enclosure was modeled using Autodesk Fusion 360 to provide mechanical support and improve the appearance of the final prototype.
